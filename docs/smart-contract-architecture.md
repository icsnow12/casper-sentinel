# Smart Contract Architecture

`contracts/casper-sentinel-governance` is a minimal Casper resolution-recorder contract.

It exists only because Phase 7A needs a real, queryable Casper Testnet transaction path for final DAO resolutions. It is not a full DAO contract and does not transfer funds.

## Storage

The contract creates a dictionary named:

```txt
resolutions
```

Each item is keyed by `proposal_id` and stores a JSON string:

```json
{
  "proposal_id": "harbor-rwa-credit",
  "project_id": "harbor-rwa-credit",
  "decision_hash": "...",
  "final_score": 74,
  "recommendation": "DILIGENCE_REQUIRED",
  "timestamp": "2026-06-19T00:00:00.000Z",
  "recorded_by": "account-hash-..."
}
```

## Entry Points

### record_resolution

Arguments:

- `proposal_id: String`
- `project_id: String`
- `decision_hash: String`
- `final_score: u8`
- `recommendation: String`
- `timestamp: String`

Validation:

- proposal ID must not be empty
- project ID must not be empty
- decision hash must not be empty
- final score must be between 0 and 100
- recommendation must not be empty
- timestamp must not be empty
- proposal ID cannot be recorded twice
- only the installer account can record resolutions

### get_resolution

Argument:

- `proposal_id: String`

Returns the JSON string for that proposal ID, or an empty string if it is not found.

### resolution_exists

Argument:

- `proposal_id: String`

Returns `true` when a dictionary entry exists.

## Ownership

On install, the contract stores `runtime::get_caller()` as `owner`. Only that account can call `record_resolution`.

Future versions can replace owner-only recording with DAO voting, multisig, or capability-based access control.

## App Integration

The Next.js app prepares a canonical payload, then submits a contract-call deploy through `casper-client` when these variables exist:

```bash
CASPER_CONTRACT_HASH
CASPER_ACCOUNT_PUBLIC_KEY
CASPER_SECRET_KEY_PATH
```

If the variables are missing, the app returns demo proof mode and does not claim real on-chain confirmation.

## Security Notes

- Do not commit private keys.
- Do not store seed phrases.
- Do not put `CASPER_SECRET_KEY_PATH` into a public deployment unless that environment is designed for secure server-side signing.
- Duplicate proposal IDs are rejected to prevent accidental overwrite.
- The decision hash should be the SHA-256 hash generated by Casper Sentinel's final resolution payload.
